# Widget API Spending — Техническое задание

## Продукт

Кроссплатформенный виджет для мониторинга расходов на AI API в реальном времени.
Показывает, сколько токенов и денег уходит на каждого провайдера, с разбивкой
по моделям и временным периодам. Конвертирует в любую валюту.

## Проблема

Разработчик использует несколько AI API (Claude, GPT, Gemini и др.).
Расходы разбросаны по разным дашбордам. Нет единого места, где видно
общий расход в реальном времени — особенно в пересчёте на свою валюту.

## Источники данных

### Anthropic (приоритет 1) — отличный API
- `GET /v1/organizations/usage_report/messages` — токены, гранулярность 1 мин
- `GET /v1/organizations/cost_report` — стоимость в USD, гранулярность 1 день
- Авторизация: Admin API key (`sk-ant-admin...`)
- Задержка данных: ~5 минут
- Ограничение: нужна организация, не работает для индивидуальных аккаунтов

### OpenAI (приоритет 2) — хороший API
- `GET /v1/organization/usage/completions` — токены, гранулярность 1 мин
- `GET /v1/organization/costs` — стоимость, только 1 день
- Авторизация: Admin API key
- Для sub-day расходов: считаем из токенов * таблица цен

### Google Gemini (приоритет 3) — нет API
- Нет REST-эндпоинта для биллинга
- Стратегия: оценка из `usageMetadata` в ответах API (токены * цена)
- Помечать как "estimated" в UI

### OpenRouter (приоритет 4) — есть API
- Требует дополнительного исследования конкретных эндпоинтов

## Архитектура

### Монорепо (pnpm + Turborepo)

```
widget-api-spending/
  packages/
    core/               — TypeScript: провайдеры, расчёт, валюты
    chrome-extension/   — Manifest V3
    desktop/            — Tauri v2
    mobile/             — React Native (позже)
```

### Сбор данных (два метода)

**Метод A — Billing API polling (основной):**
- Опрос Usage/Cost API Anthropic и OpenAI через Admin ключи
- Гранулярность: 1 мин (токены), 1 день (стоимость)
- Для sub-day стоимости: токены * локальная таблица цен

**Метод B — Перехват fetch (дополнительный):**
- Content script monkey-patches `fetch()` в браузере
- Извлекает `usage` из ответов API
- Работает без Admin ключей, но только для браузерных вызовов

### Хранение

- Chrome: IndexedDB (история), chrome.storage (настройки)
- Desktop: SQLite (история), OS keychain (ключи)
- API ключи — только в зашифрованном хранилище, никогда в plaintext

### Конвертация валют

- Frankfurter API (бесплатный, без ключа, данные ECB)
- Кэш на 24 часа, обновление раз в день

## Ключевые риски

1. **MV3 не читает response body** — решение: monkey-patch fetch через content script
2. **Admin ключи** — не все пользователи имеют org-аккаунт — делать оба метода
3. **Streaming** — usage в последнем chunk, нужен парсинг SSE
4. **Таблица цен устаревает** — хостить JSON на GitHub Pages, обновлять daily

## Не в скоупе MVP

- Кросс-девайс синхронизация
